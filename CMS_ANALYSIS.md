## 1. SiteHeader — رأس الصفحة

- **النص الأول:** `الثروة كابيتال كابيتال`
- **النص الثاني (الإنجليزي):** `Rasekhoon Capital`
- **الرمز:** حرف `ر` أبيض على خلفية ذهبية متدرجة
- **الرابط:** يوجّه إلى `/`

### العناصر المكتشفة

#### أ. الشارة العلوية (Badge)
- نقطة خضراء متحركة + نص: `شركة استثمارية مرخصة • منذ 2010`

- **السطر 2:** `بأيدي خبراء`
- **السطر 3 (أصغر حجماً):** `نحو مستقبل مالي أكثر ثباتاً`

#### ج. فقرة الوصف
```
ندير استثماراتك في أسواق الأسهم العربية والعالمية، العملات الرقمية،
المعادن النفيسة والنفط، بخبرة احترافية تمتد لأكثر من خمسة عشر عاماً.
```

## 23. صفحة تفاصيل المقال

### وصف عام
صفحة ديناميكية `article/$slug`. تعرض المقال الكامل مع قسم CTA في النهاية.

### العناصر المكتشفة

#### أ. رأس المقال
- رابط العودة: `← العودة إلى الأخبار`
- التصنيف + التاريخ + وقت القراءة
- العنوان H1
- اسم المؤلف + مسماه + زر مشاركة

#### ب. مؤشر التقدم في القراءة
- شريط تقدم أفقي (حالياً ثابت عند 30% — يحتاج تفعيلاً)

#### ج. صندوق الفقرة التمهيدية

- نص المقدمة بخط سميك

#### د. المحتوى الكامل
- أقسام H2 بنص ذهبي
- فقرات نصية

#### هـ. قسم CTA النهائي
- `هل تريد تطبيق هذه الرؤى على محفظتك؟`
- فقرة + زر `احجز استشارة مجانية`

## 25. صفحة التواصل

```
│   └── جلسات الدخول
```

## تدفق عمل المدير

### سيناريو 1: إضافة مقال جديد

```
1. لوحة الإدارة → المحتوى → المقالات
2. "إضافة مقال جديد"
3. كتابة العنوان → يُولَّد الـslug تلقائياً
4. اختيار التصنيف والمؤلف
5. كتابة المقتطف والمقدمة
6. كتابة المحتوى (محرر Markdown)
7. رفع الصورة المميزة
8. تحديد وقت القراءة
9. اختيار: مسودة / منشور
10. حفظ → يظهر فوراً في /news
```

### سيناريو 2: تحديث أسعار الأسواق

```
1. الأسواق → بيانات الأصول
2. اختيار التصنيف (أسهم/رقمية...)
3. تعديل السعر ونسبة التغيير للأصل
4. تحديث "آخر تحديث"
5. حفظ → يظهر فوراً في /markets و MarketsPreview
```

### سيناريو 3: إضافة شهادة عميل جديدة

```
1. المحتوى → آراء العملاء
2. "إضافة شهادة"
3. إدخال اسم العميل، الدور، المدينة
4. كتابة نص الشهادة
5. تحديد التقييم (1-5 نجوم)
6. رفع الصورة (اختياري)
7. تحديد الترتيب
8. حفظ → يظهر فوراً في السلايدر
```

### سيناريو 4: تعديل معلومات الاتصال

```
1. الإعدادات → بيانات الشركة
2. تعديل: العنوان / الهاتف / الإيميل / ساعات العمل
3. حفظ → يتحدث في: Footer + صفحة التواصل
```

### سيناريو 5: إضافة خدمة جديدة

```
1. الخدمات → إضافة خدمة جديدة
2. إدخال: ID, الأيقونة، الإيموجي، العنوان، الوصف
3. إضافة الميزات (قابلة للتكرار)
4. إدخال: العائد، المخاطر، الحد الأدنى
5. إضافة الأسئلة الشائعة
6. حفظ → تظهر في:
   - ServicesSection (الرئيسية)
   - /services
   - قائمة الميجا مينيو
   - /service/{id}
```

### سيناريو 6: معالجة رسالة واردة

```
1. الرسائل → الرسائل الواردة
2. عرض الرسالة (الاسم، الإيميل، الخدمة، الرسالة)
3. تغيير الحالة: جديد → قيد المراجعة
4. الرد عبر الإيميل من خارج المنصة
5. تغيير الحالة: قيد المراجعة → تمت المعالجة
```
